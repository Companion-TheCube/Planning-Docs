# Roadmap

This roadmap consolidates the phased product roadmap from the local `Roadmap` repo with the cloud/server prototype direction and the broader scope captured in the project plan.

## Phase 1: Launch (MVP)

Target from source roadmap: Q1 2026

### Core features

- Adjustable personality engine
- Wake-word detection
- Speech recognition
- Notifications for calendar and basic messaging
- To-do lists and reminders
- Alarm clock and Pomodoro timer
- Presence-aware idle mode
- Screen UI and navigation
- Audio engine for feedback and personality voice
- Initial USB-C peripheral API
- Mini-games such as Tetris, Pong, and text adventures

### Hardware support

- TheCube base unit
- Basic topper recognition
- Setup companion app

## Phase 2: Post-launch expansion

Target from source roadmap: Q2 2026

- Deeper personality behaviors and mood adaptation
- Home Assistant integration
- Advanced email summarization
- More games and content apps
- Developer libraries for third-party apps
- Embedded scripting support

## Phase 3: Physical expansion

Target from source roadmap: Q3 2026

- USB-connected accessories
- MiniCube SDK
- Local and remote app store improvements
- Ambient and seasonal modes

## Phase 4: Social and ecosystem features

Target from source roadmap: Q4 2026

- Inter-Cube personality interactions
- Remote Cube-to-Cube messaging
- Long-distance interaction features
- Better context-aware notifications
- Educational integration such as block-based programming support

## Phase 5: Format diversification

Target from source roadmap: 2027

- New form factors such as MiniCube, WideCube, and DoubleCube
- New hardware modules and display variants
- Modular add-ons and more advanced toppers

## Parallel and ongoing tracks

- Better LLM integration
- Community portal growth
- Sharable personality profiles
- Open SDK and app tooling
- Better documentation and contributor onboarding

## Server and cloud track

In parallel with the product roadmap, the server-side repos suggest a second delivery track:

- Stabilize auth and device registration
- Support streaming transcription and voice processing
- Separate user-facing cloud APIs from app distribution/catalog APIs
- Mature telemetry and admin/management capabilities
- Replace TODO-heavy prototypes with production-ready implementations
